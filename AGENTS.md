# Общие правила работы AI-агентов

## Работа с секретами

К секретам относятся:

- **Учетные данные и доступ:**
    - Токены подключения к API (REST, GraphQL и др.).
    - Токены авторизации (OAuth, Bearer, JWT).
    - Пароли и логины пользователей.
    - Сессионные Cookie-файлы.
- **Инфраструктурные данные:**
    - Строки подключения к базам данных (Connection Strings).
    - Ключи доступа к облачным сервисам (AWS Access Keys, GCP Service Accounts).
    - SSH-ключи (приватные) и файлы конфигурации (`~/.ssh/config`).
    - Учетные данные SMTP/почтовых сервисов.
- **Криптография:**
    - Секретные фразы (Seed-фразы), соли и мнемоники.
    - Приватные ключи шифрования (RSA, ECC и т.д.).
    - SSL/TLS сертификаты (особенно файлы `.key`).
- **Персональные и платежные данные (PII/PCI):**
    - Данные банковских карт (номер, CVV, срок действия).
    - Персональные идентификаторы (серии/номера паспортов, ИНН, СНИЛС, УНП).
- **Технические секреты:**
    - Webhook URL с токенами в параметрах.

Секреты хранится **только** в файле `.env`. Описание ключей и где они используются есть в этом же файле или в файле `.env.example`, при этом в файле `.env.example` нет значений самих ключей, они хранятся **только** в `.env`.

При написании скриптов, которым могут понадобится секреты всегда делай чтение этих значений (в скриптах) из файла `.env`. **Никогда** не сохраняй ключи "хардкодом", читай их только через файл переменных окружения.

Перед написанием скрипта составь список секретов, который тебе понадобится для работы и убедись, что у тебя есть все необходимые секреты, если вдруг ты не находишь какой-то секрет - останови разработку и потребуй, чтобы оператор указал его в переменной окружения.
